# AI 세미나 Q&A 정적 웹사이트 PRD v1.0

**대학원생 대상 AI 세미나에서 제출된 19개 질문과 사전 작성 답변을 카드형으로 전시하는 GitHub Pages 정적 웹사이트**

## 1. 프로젝트 개요

### 1.1 프로젝트 이름 및 한줄 설명

**AI 세미나 Q&A 정적 웹사이트** — 세미나에서 수집한 19개 질문을 카드 피드로 전시하고, 카드 클릭 시 질문 전문과 미리 작성된 개조식 답변을 보여주는 무빌드(build-less) 정적 사이트.

### 1.2 해결하려는 문제

| 문제 | 현재 상황 | 결과 |
|---|---|---|
| 질문/답변이 흩어져 있음 | 19개 질문이 `question.md` 텍스트로만 존재 | 학생이 관심 질문을 탐색·열람하기 어려움 |
| 답변 전달 채널 부재 | 세미나 현장 구두 답변에 의존 | 세미나 후 복습·재참조 불가, 답변 휘발 |
| 배포·운영 부담 | 서버·빌드 파이프라인이 필요한 형태는 과함 | 단발성 세미나 자료에 비해 유지비용 과다 |

### 1.3 핵심 가치 제안

- **즉시 열람 가능한 Q&A:** 카드 그리드 → 상세 전환으로 19개 질문을 빠르게 탐색·열람.
- **무빌드 GitHub Pages 배포:** 순수 HTML/CSS/JS만 사용, 빌드·서버 없이 `github.io`에 그대로 배포.
- **실무·연구용 개조식 답변:** 불릿·표 중심 400자 미만 답변으로 대학원생이 바로 활용 가능.

## 2. 사용자 정의

### 2.1 주요 사용자

| 역할 | 설명 | 주요 니즈 |
|---|---|---|
| **대학원생 (열람자)** | 세미나 참석 학생, 사이트의 주 사용자 | 관심 질문을 빠르게 찾아 답변을 읽고 실무·연구에 적용 |
| **강사 (콘텐츠 작성자)** | 질문에 답변을 작성하고 사이트를 배포 | 질문 데이터에 답변을 채워 넣고 GitHub Pages로 손쉽게 게시 |

### 2.2 사용자별 접근 방식

- **대학원생:** 사이트 접속 → 필터 탭으로 Part 선택 → 질문 카드 클릭 → 상세 뷰에서 답변 확인 → 목록 복귀. 특정 질문 URL(`#/q/3`)을 공유받아 직접 열람.
- **강사:** `data/questions.json`에 19문항 답변을 작성 → 정적 파일을 GitHub 저장소에 푸시 → GitHub Pages에 자동 게시.

### 3.1 질문·답변 데이터 사전 작성

`docs/question.md`의 19문항(질문 번호·소제목·본문)을 그대로 사용하고, 각 질문에 대한 답변을 미리 작성하여 `data/questions.json`에 저장한다. 질문 문구는 임의로 변경하지 않는다.

**답변 작성 규칙**

- 각 답변은 **400자 미만**(한글 기준).
- **개조식**(불릿·번호 목록·표)으로 작성하고 줄글(서술형 문단) 지양.
- 대학원생이 실무·연구에 바로 활용 가능한 구체적 수준.
- 사실·기술·학술 문항은 **웹검색으로 검증 후 작성** — 특히 Part 1·2의 도구/기법 문항과 Part 3의 Q13(Mechanistic Interpretability), Q14(Emergent Ability), Q16(The Bitter Lesson), Q18(Chinese Room).
- **출처/참고 링크는 답변 본문에 포함하지 않음** (웹검색은 검증 용도).
- **철학적 문항(Q12~Q19)**은 중립 나열이 아니라 **강사로서 명확한 입장 + 핵심 근거 2~3개**를 개조식으로 제시.

> 참고: 실제 19개 답변 본문 작성은 본 PRD의 범위가 아니며, 위 규칙에 따라 구현 단계에서 수행한다.

**입력**

- `docs/question.md` (19문항: Part 1 Q1~Q6 / Part 2 Q7~Q11 / Part 3 Q12~Q19)

**출력**

- `data/questions.json` (질문 + 사전 작성 답변, [6. 데이터 모델](#6-데이터-모델) 스키마 준수)

---

### 3.2 메인 뷰 (카드 그리드 + 필터 탭)

홈 화면에 Part별 섹션으로 구분된 질문 카드 그리드를 표시한다.

- 상단 필터 탭: `전체` / `Part 1` / `Part 2` / `Part 3` (pill 형태).
- Part별 섹션 헤더로 그룹화된 카드 그리드.
- 각 카드: 질문 번호(Q1 등), Part 뱃지, 질문 소제목, 질문 요약 미리보기.
- 카드 전체가 클릭 가능한 인터랙션 영역(터치 타깃 ≥ 44×44px).

**입력**: `data/questions.json`, 선택된 필터 값
**출력**: 필터에 해당하는 질문 카드 그리드(클릭 시 `#/q/N`로 이동)

---

### 3.3 상세 뷰 (해시 라우팅 + 딥링크)

카드 클릭 시 화면이 상세 뷰로 전환된다. **단일 페이지(SPA) + 해시 라우팅**으로 구현한다.

- 경로 예시: `index.html#/q/3`.
- 상세 뷰 구성: 질문 메타(Part·번호), 질문 전문, 답변(개조식 렌더링).
- "← 목록으로" 버튼과 브라우저 뒤로가기를 모두 지원.
- 특정 질문 URL(`#/q/3`)을 새로고침하거나 직접 열어도 해당 상세가 표시(딥링크).

**입력**: URL 해시(`#/q/N`)
**출력**: 해당 질문의 상세 뷰 렌더링

---

### 3.4 마크다운 → HTML 답변 렌더링

답변 본문은 마크다운(불릿·표)으로 저장하고 화면에서 HTML로 렌더링한다. 간단한 마크다운→HTML 변환 로직을 직접 구현하거나, 데이터에 이미 HTML로 저장하는 방식 모두 허용한다(외부 라이브러리 무의존).

**입력**: `answer` 필드(마크다운 또는 HTML 문자열)
**출력**: 렌더링된 답변 HTML(불릿·번호·표 포함)

---

### 3.5 반응형 레이아웃

`DESIGN.md` 브레이크포인트를 기준으로 모든 화면 폭에서 자연스럽게 동작한다.

- 데스크톱: 다단(3~4열) 카드 그리드 → 태블릿 2열 → 모바일 1열.
- 터치 타깃 최소 44×44px.

## 5. 기술 스택

> 본 프로젝트는 백엔드가 없는 순수 정적 사이트이므로, 템플릿 기본값(FastAPI/Next.js 등)을 사용하지 않고 아래와 같이 재정의한다.

| 영역 | 기술 |
|---|---|
| 마크업/스타일 | 순수 HTML5, CSS3 (`DESIGN.md` 토큰 기반) |
| 로직 | Vanilla JavaScript (해시 라우팅·필터·렌더링) |
| 데이터 | 정적 JSON 파일 (`data/questions.json`) |
| 폰트 | Pretendard(CDN) + `SF Pro Text`/`system-ui` 폴백 |
| 빌드 | 없음 (번들러·npm 의존성 없음) |
| 배포 | GitHub Pages (`github.io`, 상대 경로) |

**권장 파일 구조**

```
index.html              # 마크업 + 폰트/스타일/스크립트 로드
styles.css              # DESIGN.md 토큰 기반 스타일
app.js                  # 해시 라우팅 + 카드/상세 렌더링 + 필터
data/questions.json     # 19문항 질문 + 사전 작성 답변
```

## 6. 데이터 모델

### 6.1 `data/questions.json`

루트에 `questions` 배열을 두고, 각 항목은 한 질문을 표현한다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | number | 질문 고유 번호 (1~19), 라우팅 키(`#/q/{id}`)로 사용 |
| `part` | number | Part 번호 (1 / 2 / 3) |
| `partTitle` | string | Part 제목 (예: "실험 데이터 · 공학 해석에서의 AI 활용") |
| `title` | string | 질문 소제목 (예: "원본 데이터를 보존한 그래프 스타일링") |
| `question` | string | 질문 전문 (원문 그대로) |
| `answer` | string | 사전 작성 답변 (마크다운 또는 HTML, 400자 미만 개조식) |

**Part 구성**

| Part | 범위 | 제목 |
|---|---|---|
| Part 1 | Q1~Q6 | 실험 데이터 · 공학 해석에서의 AI 활용 |
| Part 2 | Q7~Q11 | AI 도구 운용 전략 및 워크플로우 |
| Part 3 | Q12~Q19 | AI의 본질에 대한 이론적 · 철학적 질문 |

**스키마 예시**

```json
{
  "questions": [
    {
      "id": 1,
      "part": 1,
      "partTitle": "실험 데이터 · 공학 해석에서의 AI 활용",
      "title": "원본 데이터를 보존한 그래프 스타일링",
      "question": "AI를 활용하여 실험 데이터 그래프를 ...",
      "answer": "- 핵심1 ...\n- 핵심2 ...\n\n| 구분 | 방법 |\n|---|---|\n| ... | ... |"
    }
  ]
}
```

## 7. 라우팅 설계 (해시 라우팅)

서버 API 대신 클라이언트 해시 라우팅으로 화면을 전환한다.

| 경로 | 화면 | 설명 |
|---|---|---|
| `#/` (또는 해시 없음) | 메인 뷰 | 필터 탭 + Part별 카드 그리드 |
| `#/q/{id}` | 상세 뷰 | 해당 질문 메타 + 전문 + 답변 |

**라우팅 동작 요구사항**

- 해시 변경(`hashchange`) 감지 시 해당 뷰 렌더링.
- `#/q/{id}` 직접 진입·새로고침 시에도 상세 뷰 복원(딥링크).
- 존재하지 않는 `id`는 메인 뷰로 폴백(또는 안내) 처리.
- 브라우저 뒤로가기로 메인 ↔ 상세 이동 가능.

```

        ▼
[상세: 질문 메타(Part·번호) + 질문 전문 + 개조식 답변 + ← 목록으로]
```

| global-ai-news | 본 사이트 |
|---|---|
| 카드 피드 홈 `/` | 질문 카드 그리드 (메인 뷰) |
| `ArticleCard` | 질문 카드 (질문 번호 · Part 뱃지 · 제목 · 요약 미리보기) |
| `FilterBar` | 필터 탭 (전체 / Part 1 / Part 2 / Part 3) |
| `Header` / `Footer` | 헤더("AI 세미나 Q&A") / 푸터 |
| `ArticleMeta` | 상세 뷰 질문 메타 (Part · 번호) |
| `/article/[id]` | 상세 뷰 (`#/q/{id}`) |
| `ThemeToggle` | 선택 사항 — 라이트 모드 우선, 여력 시 추가 |
| `SearchInput` / `/search`, `/admin` | 범위 제외 (구현하지 않음) |

## 9. 제약 사항 및 가정

**제약**

- **순수 HTML/CSS/JS만** 사용 — 빌드 단계·번들러·npm 의존성 없음.
- 외부 의존성은 **CDN 폰트(Pretendard)만** 허용(폰트 로컬 호스팅도 가능).
- GitHub Pages **상대 경로**로 동작(서브패스 배포 고려).
- `DESIGN.md` 토큰 준수: 단일 액센트 컬러 `#0066cc`, 캔버스 `#ffffff`/`#f5f5f7`, 본문 17px, 카드 1px hairline 보더(#e0e0e0) + radius 18px(`rounded.lg`) + padding 24px, 버튼/탭은 pill(`rounded.pill`). **UI 크롬에 그림자 사용 금지**, 두 번째 브랜드 컬러 도입 금지.
- 폰트 스택: `'Pretendard', 'SF Pro Text', system-ui, -apple-system, sans-serif`.

**가정**

- 질문은 19문항으로 고정(Part 1/2/3 구성).
- 콘텐츠와 UI 텍스트는 모두 한국어.
- 다크모드는 선택(옵션) — 라이트 모드 우선.
- 검색 기능과 admin 화면은 범위에서 제외.
