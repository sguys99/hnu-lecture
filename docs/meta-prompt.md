# 메타 프롬프트: AI 세미나 Q&A 정적 웹사이트 (GitHub Pages)

## 프롬프트 사용법

아래 코드블록의 프롬프트를 Claude Code에 입력하여 정적 웹사이트를 생성하세요.
이 프롬프트는 `docs/question.md`의 19개 질문에 대한 답변을 **미리 작성**하고,
GitHub Pages(github.io)에 그대로 배포 가능한 **순수 HTML/CSS/JS** Q&A 사이트를 만들도록 지시합니다.

---

## 메타 프롬프트

```
당신은 정적 웹 프론트엔드 개발 + 기술 콘텐츠 작성 전문가입니다.
아래 요구사항에 따라, GitHub Pages(github.io)에 그대로 배포 가능한
순수 HTML/CSS/JavaScript 기반 "AI 세미나 Q&A" 웹사이트를 만들어 주세요.
빌드 도구나 프레임워크(React/Next.js/Vite 등)는 사용하지 않습니다.

## 프로젝트 개요

- 대학원생 대상 AI 세미나에서 학생들이 제출한 19개 질문을 카드 형태로 전시합니다.
- 사용자가 질문 카드를 클릭하면 화면이 전환되어, 해당 질문 전문과 미리 작성된 답변을 보여줍니다.
- 질문 원문은 `docs/question.md`에 있습니다(총 19문항, 3개 Part). 이 파일을 입력 소스로 사용하세요.

## 입력 데이터

- `docs/question.md`: 19개 질문 (Part 1: Q1~Q6 / Part 2: Q7~Q11 / Part 3: Q12~Q19).
- 질문 번호, 소제목, 본문을 그대로 사용합니다(질문 문구를 임의로 바꾸지 마세요).

## 핵심 요구사항: 답변 사전 작성

19개 질문 전부에 대해 답변을 미리 작성하여 데이터 파일(`data/questions.json`)에 저장하세요.

작성 규칙:
- 각 답변은 **400자 미만**(한글 기준)으로 작성합니다.
- **개조식**으로 작성하여 가독성을 높입니다 — 불릿 포인트, 번호 목록, 표(table)를 적극 활용하세요.
- 줄글(서술형 문단)을 지양하고 핵심만 항목으로 정리합니다.
- 대학원생이 실무·연구에 바로 활용할 수 있는 수준으로 구체적으로 작성합니다.
- **사실·기술 정보가 필요한 문항은 반드시 웹검색으로 검증한 뒤 작성**하세요.
  특히 Part 1·2의 도구/기법 관련 문항과, Part 3의
  Q13(Mechanistic Interpretability), Q14(Emergent Ability),
  Q16(The Bitter Lesson), Q18(Chinese Room) 등 학술 개념이 포함된 문항은
  최신 연구·도구 동향을 검색해 정확성을 확보합니다.
- **출처/참고 링크는 답변 본문에 포함하지 마세요.** 웹검색은 사실 검증 용도로만 사용하고,
  답변은 깔끔한 본문만 남깁니다.
- **철학적 질문(Q12~Q19)**은 중립 나열이 아니라,
  **강사로서 명확한 입장을 택하고 간결한 근거를 제시**하는 방식으로 작성합니다.
  (예: "～라고 본다. 근거: …" 형태로 입장 + 핵심 근거 2~3개를 개조식으로.)

답변 본문은 마크다운(불릿/표)을 사용하고, 화면에서는 HTML로 렌더링되도록 처리하세요
(간단한 마크다운→HTML 변환 로직을 직접 구현하거나, 데이터에 이미 HTML로 저장해도 됩니다).

## 화면 구성 / 기능 요구사항

전체 화면 디자인과 구성은 다음 저장소를 **반드시 참고**하여 유사하게 구성하세요:
`https://github.com/sguys99/global-ai-news`
(카드 피드 홈 → 상세 페이지로 이어지는 뉴스 카드형 레이아웃)

해당 저장소의 구성 요소를 본 사이트에 다음과 같이 1:1로 대응시킵니다:

| global-ai-news | 본 사이트 |
|---|---|
| 카드 피드 홈 `/` | 질문 카드 그리드 (메인 뷰) |
| `ArticleCard` | 질문 카드 (질문 번호 · Part 뱃지 · 제목 · 요약 미리보기) |
| `FilterBar` | 필터 탭 (전체 / Part 1 / Part 2 / Part 3) |
| `Header` / `Footer` | 상단 헤더(사이트 타이틀 "AI 세미나 Q&A") / 하단 푸터 |
| `ArticleMeta` | 상세 뷰 질문 메타(Part, 질문 번호) |
| `/article/[id]` | 상세 뷰 (질문 전문 + 답변) |
| `ThemeToggle` (라이트/다크) | 선택 사항(옵션) — 라이트 모드 우선, 여력 시 추가 |
| `SearchInput` / `/search`, `/admin` | 범위 제외 (구현하지 않음) |

1. **메인 뷰 (카드 그리드)**
   - 상단에 필터 탭: `전체` / `Part 1` / `Part 2` / `Part 3`.
   - Part별 섹션 헤더로 구분된 카드 그리드.
   - 각 카드: 질문 번호(Q1 등), Part 뱃지, 질문 소제목, 질문 요약 미리보기.
   - 카드 전체가 클릭 가능한 인터랙션 영역.

2. **상세 뷰 (질문 + 답변)**
   - 카드 클릭 시 화면이 상세 뷰로 전환됩니다.
   - **단일 페이지(SPA) + 해시 라우팅**으로 구현하세요: 예) `index.html#/q/3`.
   - 상세 뷰에는 질문 메타(Part·번호), 질문 전문, 답변(개조식 렌더링)을 표시합니다.
   - "← 목록으로" 버튼과 브라우저 뒤로가기를 모두 지원합니다.
   - 특정 질문 URL(`#/q/3`)을 새로고침하거나 직접 열어도 해당 상세가 보이도록(딥링크) 처리합니다.

3. **반응형**
   - DESIGN.md breakpoints 기준: 1440 / 1068 / 833 / 640 / 480px.
   - 데스크톱: 다단(3~4열) 카드 그리드 → 태블릿 2열 → 모바일 1열.
   - 터치 타깃 최소 44×44px.

## 디자인 / 스타일 요구사항

- 위 `global-ai-news` 저장소의 화면 디자인·구성을 **반드시 참고**합니다.
- 색상·타이포·간격·radius 토큰은 프로젝트 루트의 `DESIGN.md`를 기준으로 사용하세요.
  (확인 결과 global-ai-news 저장소의 자체 DESIGN.md도 동일한 Apple 스타일 디자인 시스템이므로 두 레퍼런스가 일치합니다.)
  - 단일 액센트 컬러: `{colors.primary}` (#0066cc) — 모든 인터랙션 요소에만 사용. 두 번째 브랜드 컬러를 도입하지 마세요.
  - 캔버스: 흰색(#ffffff) / 파치먼트(#f5f5f7)로 리듬.
  - 본문 텍스트 17px, 헤드라인은 음수 letter-spacing(Apple tight).
  - 그림자 최소화: UI 크롬(카드/버튼/텍스트)에 그림자를 쓰지 마세요.
  - 질문 카드: 흰 배경 + 1px hairline 보더(#e0e0e0) + radius 18px(`{rounded.lg}`) + padding 24px.
  - 버튼/탭은 pill(`{rounded.pill}`) 그래머를 따릅니다.
- **폰트**:
  - 한글 폰트는 **Pretendard**를 사용합니다(CDN 링크로 로드).
    예: `https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/static/pretendard.css`
  - 영문/디스플레이 폰트는 DESIGN.md의 폰트 스택을 따릅니다
    (`SF Pro Display`/`SF Pro Text` → `system-ui, -apple-system, sans-serif` 폴백).
  - 최종 font-family 스택 예: `'Pretendard', 'SF Pro Text', system-ui, -apple-system, sans-serif`.

## 기술 제약 (고정)

- **순수 HTML/CSS/JavaScript만** 사용. 빌드 단계, 번들러, npm 의존성 없음.
- 외부 의존성은 **CDN 폰트(Pretendard)만** 허용. (선택적으로 폰트 로컬 호스팅도 가능)
- GitHub Pages에서 **상대 경로**로 동작해야 합니다(저장소 서브패스 배포 `username.github.io/repo/` 고려).
- 권장 파일 구조:
  ```
  index.html              # 마크업 + 폰트/스타일/스크립트 로드
  styles.css              # DESIGN.md 토큰 기반 스타일
  app.js                  # 해시 라우팅 + 카드/상세 렌더링 + 필터
  data/questions.json     # 19문항 질문 + 사전 작성한 답변
  ```
- `data/questions.json` 스키마 예시:
  ```json
  {
    "questions": [
      {
        "id": 1,
        "part": 1,
        "partTitle": "실험 데이터 · 공학 해석에서의 AI 활용",
        "title": "원본 데이터를 보존한 그래프 스타일링",
        "question": "AI를 활용하여 실험 데이터 그래프를 ...",
        "answer": "- 핵심1 ...\n- 핵심2 ...\n\n| 구분 | 방법 |\n|---|---|\n| ... | ... |"
      }
    ]
  }
  ```

## 출력 형식

- 위 파일들(`index.html`, `styles.css`, `app.js`, `data/questions.json`)을 생성합니다.
- `data/questions.json`에는 19문항 전부의 질문과 **사전 작성된 답변**이 채워져 있어야 합니다.
- 모든 콘텐츠와 UI 텍스트는 한국어로 작성합니다.
- 로컬에서 `index.html`을 열거나 정적 서버로 띄웠을 때, 카드 그리드 → 카드 클릭 → 상세 전환 →
  목록 복귀가 정상 동작해야 합니다.

위 요구사항을 모두 반영하여 사이트를 구현해 주세요.
```

## 예상 산출 구조

이 메타 프롬프트를 실행하면 다음과 같은 정적 사이트가 생성됩니다:

```
.
├── index.html              # 헤더 + 카드 그리드/상세 컨테이너 + 폰트/스크립트 로드
├── styles.css              # DESIGN.md 토큰 기반 (단일 액센트, hairline 카드, Pretendard)
├── app.js                  # 해시 라우팅(#/q/N) + 필터 탭 + 카드/상세 렌더링
└── data/
    └── questions.json      # 19문항 질문 + 400자 미만 개조식 답변
```

화면 흐름:

```
[메인: 필터 탭 + Part별 질문 카드 그리드]
        │  카드 클릭 (#/q/N)
        ▼
[상세: 질문 메타 + 질문 전문 + 개조식 답변 + ← 목록으로]
```
